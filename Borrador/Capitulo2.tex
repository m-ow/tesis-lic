\chapter{Cálculo $\lambda$}

\begin{defn}
Utilizamos una representación \textit{locally nameless} 
para el cálculo $\lambda$, donde las variables ligadas se representan 
como números naturales (índices de De Bruijn) y las variables libres como
términos del tipo \lean{Var}. El tipo \lean{Var} representa nombres: 
la igualdad es decidible en ellos (\lean{DecidableEq}) 
y es posible generar uno nuevo para cualquier conjunto finito dado
de nombres (\lean{HasFresh}).
\begin{leancode}
universe u
variable {Var : Type u} [HasFresh Var] [DecidableEq Var]

inductive Term (Var : Type u)
  | bvar : ℕ → Term Var
  | fvar : Var → Term Var
  | abs  : Term Var → Term Var
  | app  : Term Var → Term Var → Term Var
\end{leancode}
Por ejemplo, podemos representar la expresión ``$\lambda x. \, x \, y$'' de la siguiente manera.
\begin{leancode}
variable {y : Var}
def ejemplo_rep := abs (app (bvar 0) (fvar y))
\end{leancode}
Lo cual también se puede escribir como ``$\lambda . \, 0 \, y$''. 
\end{defn}

\begin{defn}
La sustitución consiste en remplazar una variable libre por un término. 
\begin{leancode}
def subst (m : Term Var) (x : Var) (sub : Term Var) : Term Var :=
  match m with
  | bvar i  => bvar i
  | fvar x' => if x = x' then sub else fvar x'
  | app l r => app (l.subst x sub) (r.subst x sub)
  | abs M   => abs <| M.subst x sub
\end{leancode}
Definimos una notación para la sustitución de variables libres que imita la notación matemática estándar.
\begin{leancode}
scoped notation t:max "[" x ":=" t' "]" => subst t x t'
\end{leancode}
\end{defn}

\begin{defn}
La función \lean{fv}, definida a continuación, 
calcula el conjunto de variables libres de una expresión. 
Dado que utilizamos una representación \textit{locally nameless}, 
donde las variables ligadas se representan como índices, 
cualquier nombre que veamos es una variable libre de un término.
\begin{leancode}
def fv : Term Var → Finset Var
  | bvar _ => {}
  | fvar x => {x}
  | abs e1 => e1.fv
  | app l r => l.fv ∪ r.fv
\end{leancode}
El tipo \lean{Finset Var} representa un conjunto finito 
de elementos del tipo \lean{Var}. 
\end{defn}

\begin{defn}
La operación de apertura sustituye un índice por un término. 
\begin{leancode}
def openRec (i : ℕ) (sub : Term Var) : Term Var → Term Var
  | bvar i' => if i = i' then sub else bvar i'
  | fvar x  => fvar x
  | app l r => app (openRec i sub l) (openRec i sub r)
  | abs M   => abs <| openRec (i+1) sub M
\end{leancode}
También definimos una notación para \lean{openRec}.
\begin{leancode}
scoped notation:68 e "⟦" i " ↝ " sub "⟧" => openRec i sub e
\end{leancode}
Las aplicaciones habituales de esta función sustituyen el índice 
cero por una variable o un término $\lambda$. La siguiente definición 
es una forma práctica de indicar estos usos.
\begin{leancode}
def open' {X} (e u) := @openRec X 0 u e  
 
scoped infixr:80 " ^ " => open'
\end{leancode}
Así, \lean{(e ^ x)} puede leerse como 
``reemplaza el índice $0$ por la variable \lean{x} en \lean{e}''
o ``abre \lean{e} con la variable \lean{x}''. 
\end{defn}

\begin{defn}[\textit{Local closure}]
Un término se denomina ``localmente cerrado''
cuando no contiene índices no ligados.
La proposición \lean{LC e} se cumple cuando una expresión \lean{e} 
es localmente cerrada.
\begin{leancode}
inductive LC : Term Var → Prop
  | fvar (x : Var)  : LC (fvar x)
  | abs (L : Finset Var) (e : Term Var) : (∀ x ∉ L, LC (e ^ fvar x)) → LC (abs e)
  | app {l r} : l.LC → r.LC → LC (app l r)
\end{leancode}
\end{defn}

\begin{defn}
Dada una relación \lean{r} en el tipo \lean{Term Var},
se llama cerradura compatible de \lean{r} a la relación \lean{CompatGen r},
definida de la siguiente manera.
\begin{leancode}
inductive CompatGen (r : Term Var → Term Var → Prop) : Term Var → Term Var → Prop
  | base : r M N → CompatGen r M N
  | appL : LC Z → CompatGen r M N → CompatGen r (app M Z) (app N Z)
  | appR: LC Z → CompatGen r M N → CompatGen r (app Z M) (app Z N)
  | xi (L : Finset Var) : (∀ x ∉ L, CompatGen r (M ^ fvar x) (N ^ fvar x)) →
    CompatGen r (abs M) (abs N)
\end{leancode}
\end{defn}

\begin{defn}
Sea \lean{Beta} una relación en \lean{Term Var}, definida de la siguiente manera.
\begin{leancode}
inductive Beta : Term Var → Term Var → Prop
  | beta : LC (abs A) → LC B → Beta (app (abs A) B) (A ^ B)
\end{leancode}

\begin{enumerate}[(i)]
\item \textit{La reducción $\beta$ de un solo paso} es la cerradura compatible de \lean{Beta}.
\begin{leancode}
@[reduction_sys "βᶠ"]
abbrev FullBeta : Term Var → Term Var → Prop := CompatGen Beta
\end{leancode}
\item \textit{La reducción $\beta$} es la cerradura reflexiva y transitiva de \lean{CompatGen Beta},
es decir, \lean{ReflTransGen FullBeta}.
\end{enumerate}

\end{defn}

\begin{defn}
Sea \lean{Eta} una relación en \lean{Term Var}, definida de la siguiente manera.
\begin{leancode}
inductive Eta : Term Var → Term Var → Prop
  | eta : LC M → Eta (abs (app M (bvar 0))) M
\end{leancode}

\begin{enumerate}[(i)]
\item \textit{La reducción $\eta$ de un solo paso} es la cerradura compatible de \lean{Eta}.
\begin{leancode}
@[reduction_sys "ηᶠ"]
abbrev FullEta : Term Var → Term Var → Prop := CompatGen Eta
\end{leancode}
\item \textit{La reducción $\eta$} es la cerradura reflexiva y transitiva de \lean{CompatGen Eta},
esto es \lean{ReflTransGen FullEta}.
\end{enumerate}

\end{defn}
